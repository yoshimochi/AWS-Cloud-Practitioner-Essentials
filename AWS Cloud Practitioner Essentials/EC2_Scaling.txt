# EC2のスケーリング①
## スケーラビリティによって、需要の変化に応じて自動的にスケールアウトまたはスケールインするアーキテクチャを設計できるため、実際に使用したリソースに対してのみ料金が発生する。
## EC2インスタンスにスケーラビリティを提供するサービスが、Amazon EC2 Auto Scaling

# Amazon EC2 Auto Scaling
## アプリケーションの需要の変化に応じてEC2インスタンスを自動的に作成または削除することができる。必要に応じて EC2 インスタンスを自動的にスケールインおよびスケールアウトすることにより、アプリケーションの可用性をさらに高めることができます。

## スケーリングの種類
### 動的スケーリング・・・変化する需要に対応する
### 予測スケーリング・・・予測された需要に基づいて、適切な数のEC２インスタンスを自動的に用意する


# EC2のスケーリング②
# Amazon EC2 Auto Scaling
## Amazon EC2 Auto Scalingを組み込むと必要に応じて新しいインスタンスを追加したり、不要になった場合は削除したりできる。
## Auto Scaling グループのサイズの設定
### ①EC2インスタンスの最小数を１から設定できる。
### ②必要キャパシティーとして、宰相が１でも２つのEC2インスタンスを設定することができる。
### ③需要の増加に応じてスケールアウトするよう設定するために、最大キャパシティーを設定できる

# Amazon EC2 Auto Scaling は Amazon EC2 インスタンスを使用するため、料金は使用したインスタンスに対してのみ発生します。これで、費用を削減しながら優れたカスタマーエクスペリエンスを提供できる、コスト効率の良いアーキテクチャを構築することができます。
