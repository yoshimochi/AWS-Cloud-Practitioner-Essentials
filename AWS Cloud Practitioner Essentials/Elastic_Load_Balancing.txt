# Elastic Load Balancing
## Elastic Load Balancing は、アプリケーションへのトラフィックを、EC2 インスタンスなどのリソースへ自動的に分散する AWS のサービスです。
## ロードバランサーは、Auto Scalingグループのインスタンスに対する通信の代表的な通信先として機能します。つまり、受信トラフィックの量に応じてEC2インスタンスを増減させても、まず初めにロードバランサーがリクエストを受け取り、その後EC2インスタンスへリクエストが分散される。
## ロードバランサーがあることで、１つのインスタンスに大きな負荷がかからないようになる
## Elastic Load Balancing と Amazon EC2 Auto Scaling は異なるサービスですが、この 2 つのサービスを連動させることにより、Amazon EC2 で実行中のアプリケーションの高いパフォーマンスと可用性が実現できる。

## コーヒーショップの例
### 需要が高い期間
### Auto Scaling・・・客数の増加に合わせて新たなレジ（EC2インスタンス）を開けて対応する
### Load Balancing・・・客の注文を均等に分散し、開いているレジ全体で対応できるように、適切なレジへ客を案内する
